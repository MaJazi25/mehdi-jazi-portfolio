Status Reports

The Contractor Connect status reports were used to monitor project progress, evaluate team performance, and identify areas that needed attention during development. In Status Report 1, the project health was marked as Yellow (Warning), which showed that the project was moving forward but still had minor delays related to integration work and team scheduling.

During this reporting period, the team completed several important setup and foundation tasks. These included creating the GitHub repository, setting up the CodeIgniter 4 environment, securing the ContractorConnect.cc domain, configuring cPanel hosting, preparing the database schema for front-end integration, and completing a presentable home page. Team communication was also organized through WhatsApp and Microsoft Teams.

Some planned work remained unfinished at this stage, including full CRUD implementation, completion of role-based access control, front-end and back-end integration, comprehensive testing, and deployment of a fully functional live version. Because of this, the team set clear goals for the next reporting period to improve overall project health and return to Green.

Upcoming Objectives

  • Complete core backend functionality, including CRUD operations and role-based access control.
  • Finalize backend database implementation to support application features.
  • Develop responsive front-end interfaces using reusable components.
  • Integrate front-end views with backend controllers and routes.
  • Implement secure user authentication and authorization.
  • Perform API testing and validation for stability.
  • Prepare the system for an end-to-end functional demonstration.

My Role and Planned Work

As part of the development work for the next period, I was assigned to complete the backend database implementation. I was also assigned, together with another team member, to support backend CRUD operations, database integration, authentication, and role-based access control milestones.

Issues and Risks

The report identified several challenges and risks, including possible team availability issues, delays in front-end and back-end integration, bugs in CRUD and authentication features, and the chance of security or input validation issues being found late in development. To reduce these risks, the team planned regular check-ins, task reassignment when needed, early Postman testing, peer code reviews, incremental development, and early input validation.
